0. Задача 
  * считать описание вакансии
  * подсчитать BaseScoring соответствия данной вакансии нашему базовому резюме сооискателя. (от 0 до 100%).
  * вычленить из описания вакансии формальниые критерии, по которым будет проводится дальнейшая оценка.
  * подсчитать из найденных формальных критериев AdditionalScoring (также приведенный к шкале 0..100%)
  * подсчитать SummaryScoring как средневзвешенное BaseScoring и AdditionalScoring (веса BASE_SCORING_WEIGHT и ADDITIONAL_SCORING_WEIGHT в .env), шкала итога 0..100. Все три значения внести в таблицу.
  * из всех обработанных вакансий отбирать только превышающие SummaryScoring > Min указанного в .env
  *  
1. Вся работа проходит на уровне одного GoogleSheet. 
2. список URL для обработки расположен в колонке URL листа Tracker
3. Основной критерий обработки - поле URL заполнено, а поле Description нет.
4. Считанное описание вакансии в виде текста поместить в ячейку Description.
5. Рассчетные значения BaseScoring, AdditionalScoring, SummaryScoring поместить в колонки с аналогичными названиями.
6. При расчете AdditionalScoring руководствоваться следующими принципами: Итоговое значение должно находится 
   в интервале от 0 до 100. Список дополнительных критериев, по которым будет оцениваться вакансия лежит 
   на листе Additional Filter. Первая строка - заголовки. Колонка 1 - название критерия (например, "размер компании")
   Колонка 2 - базовая оценка (вес) данного критерия (например 1 балл, значит критерий может иметь от 0 до 1 балла). 
   Колонка 3 - пояснение как получать численное значение критерия. (например, размер компании: до 50 человек = 0.2, 
   от 51 до 1000 = 0.8, более 1000 = 1).
7. Правила расчета BaseScoring определяются по полноте соответствия базового резюме соискателя 
   требованиям вакансии (т.е. все требования вакансии мы принимаем за 100%). Как конкретно подсчитать 
   придется придумать ИИ. Вероятно сначала надо определить общее количество различных требований в вакансии. 
   Это самое простое, возможно некоторые требования будут стоить дороже, чем другие. Но об этом надо подумать.

10. Уточнения к задаче "Подготовка резюме".
  10.1. Адаптированное резюме должно формироваться в формате GoogleDoc, по образцу файла CV_Template 
  10.2. Данный файл должен рассматриваться только как образец оформления, структура документа, фотографии 
  должны быть перенесены. В то же время, наполнение текстом должно быть на основе 
  адаптированного резюме.
  10.3. Структура исходных данных:
      Лист "Master CV" - колонка A - разделы, колонка B - содержание.
      Раздел "Master CV" - текст базового резюме сооискателя
      Раздел "CV Doc Template" - ссылка на GoogleDoc, содержащий шаблон/образец оформления резюме.
      Раздел "System_prompt" - текст системного промпта LLM для формирования адаптированного резюме. 
  10.3. Текст адаптированного резюме должен готовится на основе текста, размещенного на листе "Master CV" в ячейче A4.
  10.4. LLM должен провести адаптацию резюме к требованиям вакансии согласно требованиям системного промпта.
  10.5 Готовый файл адаптированного резюме в формате GoogleDoc должен быть размещен в специальной папке Adapted_CVs 
       на GoogleDrive. Ссылка на этот файл помещается в колонку "New CV File" на листе Tracker и означает окончание 
       работы с этой строкой.
