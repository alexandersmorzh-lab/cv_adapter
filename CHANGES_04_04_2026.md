# Изменения от 04.04.2026 — Переработка потока анализа

## Обзор изменений

Реализован новый поток обработки вакансий согласно требованиям в `Analyzer.txt` (раздел 8).

### Главные отличия

**Было (старый поток):**
- Запуск с колонки URL листа Tracker
- Загрузка страницы по ссылке
- Поиск Description на странице
- Расчёт скоринга в Tracker
- Адаптация CV в отдельном режиме

**Стало (новый поток):**
- Запуск с листа Search DataBase
- Description уже заполнен (не нужно скачивать)
- Фильтр WrongPhrases (запрещённые фразы)
- Расчёт скоринга в Search DataBase
- Автоматическое добавление в Tracker (при SummaryScoring > MIN)
- Адаптация CV запускается отдельно

## Детали реализации

### 1. Конфигурация (config.py)

Добавлены новые переменные:
```python
SHEET_SEARCH_DATABASE = "Search DataBase"
SHEET_WRONG_PHRASES = "WrongPhrases"
COL_WRONG_PHRASES = "WrongPhrases"        # новая колонка в Search DataBase
COL_TRACKER_ID = "TrackerID"              # новая колонка в Search DataBase
```

### 2. Модуль работы с таблицами (sheets.py)

Добавлены 6 функций:

#### `read_wrong_phrases(client)` ⭐
Читает запрещённые фразы из листа WrongPhrases (колонка А).
```python
phrases = sheets.read_wrong_phrases(client)
# Возвращает: ["Python", "Java", ...]
```

#### `get_search_database_rows(client)`
Читает строки из Search DataBase для обработки:
- Description заполнен
- SummaryScoring пуст

Возвращает: `(worksheet, rows, col_indices, headers)`

#### `write_search_database_result(worksheet, row_num, col_indices, ...)`
Записывает результаты анализа (баллы и флаг WrongPhrases) в Search DataBase.

#### `get_next_tracker_id(client)`
Получает следующий доступный ID для листа Tracker.

#### `add_row_to_tracker(client, row_id, row_data)`
Добавляет новую строку в Tracker с данными из Search DataBase.

Копирует колонки:
- Timestamp, Title, Company, Location, URL, Source, Description
- BaseScoring, AdditionalScoring, SummaryScoring

#### `update_search_database_tracker_id(worksheet, row_num, col_tracker_id, tracker_id)`
Обновляет колонку TrackerID в Search DataBase (связь с Tracker).

### 3. Модуль анализа (analyzer.py)

Добавлены 2 функции:

#### `check_wrong_phrases(description, wrong_phrases)` 🔍
Проверяет, содержит ли Description хотя бы одну фразу из запрещённого списка.
```python
has_forbidden = analyzer.check_wrong_phrases(desc, ["Python", "Java"])
```

#### `run_analyzer_search_database(client, base_cv)` ⭐ ГЛАВНАЯ
Новый главный поток обработки:

```
1. Читает запрещённые фразы
2. Читает строки Search DataBase (Description есть, SummaryScoring пуст)
3. Для каждой строки:
   a) Проверяет WrongPhrases
      - Если найдена: WrongPhrases=1, все баллы=0, переход к следующей
   b) Вызывает LLM для расчёта BaseScoring, AdditionalScoring
   c) Вычисляет SummaryScoring (средневзвешенное)
   d) Записывает в Search DataBase
   e) Если SummaryScoring > MIN_SUMMARY_SCORE:
      - Добавляет строку в Tracker
      - Присваивает новый ID
      - Обновляет TrackerID в Search DataBase
   f) Применяет rate limit между запросами
```

Возвращает: `(успешно обработано, всего строк, добавлено в Tracker)`

### 4. Главный модуль (main.py)

Переработан поток main():
- Вместо 4 шагов теперь 3 шага
- Вызывает новую функцию `run_analyzer_search_database()`
- Адаптация CV теперь отдельная операция (не включена в main flow)

## Использование

### Требуемая структура Google Sheets

Обязательные листы:
- **Master CV** — базовое резюме
- **Search DataBase** — входные данные для анализа (Description + поля результата)
- **WrongPhrases** — список запрещённых фраз (колонка A)
- **Additional Filter** — дополнительные критерии оценки
- **Tracker** — финальные результаты

### Требуемые колонки

**Search DataBase:**
- Description (текст вакансии) ✓
- BaseScoring (результат)
- AdditionalScoring (результат)
- SummaryScoring (результат)
- WrongPhrases (флаг 0/1)
- TrackerID (ID в Tracker)

**Tracker:**
- ID (уникальный)
- TimestampTitle, Company, Location, URL, Source
- Description
- BaseScoring, AdditionalScoring, SummaryScoring
- Adapted CV (для адаптации)

### Запуск

```python
python main.py
```

Или через exe (если скомпилировано):
```
CVAdapter.exe
```

```
  ⚠ Найдена запрещённая фраза → WrongPhrases=1, скоринг=0

  → SummaryScoring=81.2 > 50.0 → добавляю в Tracker
  ✓ Добавлено в Tracker с ID=42
```

## Обработка ошибок

- Если лист не найден → FATAL ошибка
- Если LLM не отвечает → выводит ошибку, пропускает строку
- Если не удалось добавить в Tracker → выводит warning, продолжает
- Все ошибки логируются в DEBUG режиме

## Совместимость

- Старая функция `run_analyzer()` оставлена для обратной совместимости
- Можно вместо нового потока использовать `run_analyzer()` если нужно обрабатывать URL
- Оба потока работают независимо
