1. На основе обкатанного механизма и алгоритмов приложения CV_Adapter надо разработать
новое приложение, ориентированное на множество соискателей, каждый со своими критериями поиска вакансий.
Новое приложение должно быть ориентировано на работу в автоматическом режиме с заданной периодичностью.

2. Приложение должно содержать все данные в своей БД MySQL. 
3. Использование GoogleSheet не предусматривается.
4. В приложении предсматриваются следующие логические части:
  а. блок Соискатель. Куда входят занесение и хранение исходных данных для поиска вакансий:
     базовое резюме - файл в одном из читаемых форматов (GoogleDoc, MS DOC, txt, rtf)
     критерии первичного поиска вакансий во внешних системах: должность, локация, отрасли, время от подачи, активная?, тип занятости (в офисе, удаленная, гибрид), опыт (лет)
     системный промпт (*) - текст для переопределения промпта оценки BaseScoring
     системный промпт (*) - текст для переопределения промпта адаптации резюме
     критерии дополнительной оценки вакансии (размер компании, число подавшихся соискателей и т.п.)
     недопустимые слова - набор ключевых слов для вторичной фильтрации вакансий по вхождению в наименование или описание вакансии.
     правило отбора - <N в день и >M баллов
  б. блок Сбор вакансий. По заданным критериям первичного поиска подключается ко внешним системам
     и осущствляет поиск и загрузку вакансий в БД.
  в. блок Анализа. Загруженные вакансии анализируются и получают следующие дополнительные признаки:
     * наличие недопустимых слов
     * BaseScoring - оценка от 0 до 100 баллов соответствия резюме требованиям вакансии (LLM)
     * BaseScoringReason - текст обоснование от LLM оценки
     * AddScoring - оценка от 0 до 100 баллов соответствия вакансии дополнительным требованиям соискателя
     * AddScoringReason - текст обоснование от LLM оценки
     * SummaryScoring - итоговая оценка вакансии для соискателя
  г. блок коммуникации с соискателем. Отправить требуемым способом отобранные вакансии и запросить 
     нужно ли под них готовить адаптированное резюме.
  д. блок Адаптации резюме.
  е. блок Управления. запуск по расписанию других блоков
