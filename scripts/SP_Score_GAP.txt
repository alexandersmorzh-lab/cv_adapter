Objective: Conduct a dual-layered evaluation of a candidate's resume against a specific Job Description (JD). Provide a highly scannable, human-centric screening gap analysis alongside an objective, mathematically rigorous JSON scoring matrix.

Inputs: 
- Job Description (JD)
- Resume/CV Text

---

### Phase 1: Critical Filters & Algorithmic Scoring Rules
1. Language Disqualifier: If the JD text is written in any language other than English, OR if the requirements explicitly state that languages other than Russian and English are required, the final total_score must immediately be set to 0, and the recommendation must be "reject".
2. Objective Facts Only: Analyze only explicit facts. If a skill is not mentioned in the resume, it does not exist. Do not assume or infer unstated experience.
3. Must-Have vs. Nice-to-Have: Categorize JD requirements into "Must-have" and "Nice-to-have" for each scoring category.
   - If a "Must-have" requirement in a category is missing, the score for that category cannot exceed 50.
   - "Nice-to-have" requirements can add up to 20 bonus points to a category, but ONLY if all "Must-have" requirements for that category are met.
4. Missing JD Data: If the JD lists no requirements for a specific category, set score: 100 and weight: 0.0 to ensure mathematical neutrality.
5. Weighting Matrix:
   - experience (0.35): Relevance of roles, years of experience, industry, and level of responsibility.
   - hard_skills (0.30): Tech stack, key tools, and professional certifications/technologies.
   - education (0.10): Degree level, field of study, and relevant diplomas.
   - soft_skills (0.10): Communication, leadership, teamwork, and work style.
   - seniority (0.10): Match between required seniority (Jr/Mid/Sr/Lead) and actual track record.
   - additional (0.05): Languages, location/relocation, specific work permits, or travel readiness.
